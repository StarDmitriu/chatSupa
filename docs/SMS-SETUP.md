# Настройка отправки SMS кодов

## Проблема

При попытке получить код подтверждения SMS не отправляется. В логах видно:
```
WARN [SmsService] SMSRU_API_ID не задан -> DEV mode: SMS не отправляем
```

## Решение

### 1. Получить API ключ от sms.ru

1. Зарегистрируйтесь на https://sms.ru
2. Перейдите в раздел API: https://sms.ru/?panel=api
3. Создайте API ID и скопируйте его

### 2. Добавить ключ в .env

Откройте `/var/www/.env` и добавьте:

```env
SMSRU_API_ID=ваш_api_id_здесь
SMSRU_FROM=Chatrassylka  # опционально: имя отправителя
```

### 3. Скопировать в backend/.env

```bash
cp /var/www/.env /var/www/backend/.env
```

### 4. Перезапустить backend

```bash
pm2 restart backend
```

## Проверка

После настройки попробуйте запросить код снова. В логах должно появиться:
```
LOG [SmsService] SMS sent via sms.ru to +79857040111
```

Вместо предупреждения о DEV mode.

## Альтернативные SMS провайдеры

Если нужно использовать другой SMS провайдер, нужно изменить код в `/var/www/backend/src/sms/sms.service.ts` для работы с другим API.

## Dev режим (без API ключа)

Если `SMSRU_API_ID` не задан, приложение работает в dev режиме:
- Код генерируется и сохраняется в БД
- SMS не отправляется
- В логах видно код, который был бы отправлен (для тестирования)

**Важно:** В production обязательно настройте реальный SMS провайдер!
