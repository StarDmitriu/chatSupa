# Пароль админ-панели

Админ-панель (`/admin`) защищена дополнительным паролем. Данные (таблица пользователей) показываются **только после** успешного ввода пароля.

## Где задаётся пароль

Пароль задаётся **на сервере** в переменной окружения бэкенда:

- **Имя переменной:** `ADMIN_PANEL_PASSWORD`
- **Где прописать:** в `backend/.env` (или в конфиге PM2/системы, если env задаётся там).

Пример в `backend/.env`:

```env
ADMIN_PANEL_PASSWORD=ваш_секретный_пароль
```

После изменения переменной нужно перезапустить бэкенд, например:

```bash
pm2 restart backend
```

## Поведение

- Если **`ADMIN_PANEL_PASSWORD` не задан** — бэкенд не требует пароль для маршрутов `/admin/*` (остаётся только проверка, что пользователь — admin по JWT).
- Если **переменная задана** — без заголовка `X-Admin-Password` с верным значением запросы к `/admin/*` возвращают 403 (`admin_password_required` или `admin_password_invalid`).

На фронте при первом заходе на `/admin` показывается экран ввода пароля; после успешной проверки пароль сохраняется в `localStorage` и данные отображаются. Сменить пароль в браузере можно по кнопке «Пароль: задан» в шапке админки.
