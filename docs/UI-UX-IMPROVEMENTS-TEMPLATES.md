# Анализ UI/UX экранов шаблонов и выбор групп

## 🔍 Выявленные проблемы

### 1. **Дублирование функционала выбора групп**

**Проблема:**
- Выбор групп происходит в **двух местах**:
  1. На странице `/dashboard/groups` и `/dashboard/telegram-groups` (общий выбор)
  2. На странице `/dashboard/templates` (выбор для шаблонов)
  3. На странице редактирования шаблона `/dashboard/templates/[id]` (выбор для конкретного шаблона)

**Проблемы:**
- Пользователь не понимает, где именно выбирать группы
- Два разных интерфейса для одной задачи
- На странице шаблонов выбор групп происходит в правой колонке, что неочевидно

### 2. **Сложная структура страницы шаблонов**

**Проблема:**
- Двухколоночная структура (левая - шаблоны, правая - выбор групп)
- Много текста-инструкций
- Непонятно, что выбор групп на этой странице влияет на все шаблоны

**Код:**
```tsx
// templates/page.tsx - строка 591-595
<div style={{ fontSize: 12, opacity: 0.85, marginBottom: 8 }}>
  Сначала выберите группы (галочками), в которые можно отправлять шаблоны. 
  Эти группы будут доступны при создании и редактировании шаблонов.
</div>
```

### 3. **Неочевидный процесс редактирования шаблона**

**Проблема:**
- Нужно **два шага**: сначала выбрать группы, потом сохранить шаблон
- Кнопка "Сохранить группы (WA)" отдельно от "Сохранить шаблон"
- Предупреждение о том, что группы не выбраны появляется только после переключения канала

**Код:**
```tsx
// [templateId]/page.tsx - строка 598-616
{groups.length > 0 && selectedGroupJids.length === 0 && (
  <div style={{...}}>
    Для канала <b>{channel.toUpperCase()}</b> не выбраны группы.
    <br />
    Рассылка по этому каналу не запустится...
  </div>
)}
```

### 4. **Несогласованность стилей**

**Проблема:**
- Страницы групп (`groups/page.tsx`, `telegram-groups/page.tsx`) используют модульные CSS (`groups.module.css`)
- Страницы шаблонов используют глобальные классы (`page.css`)
- Разные стили кнопок и таблиц

### 5. **Неудобная навигация**

**Проблема:**
- На странице шаблонов есть кнопка "2. Запустить рассылку →", но нет явной ссылки на группы
- Нужно переключаться между WA и TG вручную
- Не видно, сколько групп выбрано для каждого канала

## 🎯 Предложения по улучшению

### 1. **Упростить выбор групп**

**Решение:**
- Убрать выбор групп со страницы шаблонов
- Оставить выбор только на страницах `/dashboard/groups` и `/dashboard/telegram-groups`
- При создании/редактировании шаблона показывать только **уже выбранные** группы
- Добавить ссылку "Выбрать группы →" если групп нет

**Изменения:**
```tsx
// templates/page.tsx - убрать правую колонку с выбором групп
// Вместо этого добавить:
<div className='tpl-info'>
  <p>Выбранные группы используются для всех шаблонов.</p>
  <Link href="/dashboard/groups">Управление группами WhatsApp →</Link>
  <Link href="/dashboard/telegram-groups">Управление группами Telegram →</Link>
</div>
```

### 2. **Упростить страницу редактирования шаблона**

**Решение:**
- Автоматически сохранять группы при сохранении шаблона (без отдельной кнопки)
- Показывать предупреждение сразу, если нет выбранных групп
- Добавить визуальную индикацию: сколько групп выбрано для WA/TG

**Изменения:**
```tsx
// [templateId]/page.tsx
// Убрать кнопку "Сохранить группы (WA/TG)"
// Сохранять группы автоматически в onSave
const onSave = async (values: any) => {
  // ... сохранение шаблона
  // Автоматически сохраняем группы
  await saveTargetsForTemplate(templateId)
}
```

### 3. **Унифицировать стили**

**Решение:**
- Использовать единый стиль кнопок из `globals.css`
- Применить желтые карточки (#F8F1C9) для контейнеров групп
- Использовать одинаковые скругления (26px для карточек, 999px для кнопок)

### 4. **Улучшить визуальную иерархию**

**Решение:**
- Уменьшить размер заголовка "Ваши шаблоны" (сейчас 56px → сделать 36px)
- Добавить визуальное разделение между шаблонами
- Показывать статус выбранных групп в шапке страницы

**Пример:**
```tsx
<div className='tpl-header'>
  <h1>Ваши шаблоны</h1>
  <div className='tpl-status'>
    <span>WA: {waSelectedCount} групп</span>
    <span>TG: {tgSelectedCount} групп</span>
  </div>
</div>
```

### 5. **Добавить подсказки и валидацию**

**Решение:**
- Показывать предупреждение при создании шаблона, если нет выбранных групп
- Добавить tooltip с объяснением, что группы выбираются отдельно
- Валидировать перед сохранением: есть ли выбранные группы

### 6. **Упростить создание шаблона**

**Решение:**
- Объединить выбор WA и TG групп в один интерфейс с табами
- Показывать предпросмотр: сколько групп будет использовано
- Автоматически выбирать все группы по умолчанию (можно снять)
