# Интеграции: Google Таблица и бэкап шаблонов

## Где это в интерфейсе

- **Кабинет → Интеграции:** таблица шаблонов (создать/ссылка), **Скачать бэкап**, **Восстановить из таблицы**.
- **Дашборд → Шаблоны:** блок «Снимок шаблонов (бэкап)» — **Скачать бэкап**, **Восстановить из файла** (CSV), **Восстановить из таблицы**.

Таблица и CSV используются как **полный снимок шаблонов** (все поля: название, текст, медиа, скорость, время и т.д.). Контакты/группы задаются в «Группы WA» и «Группы TG».

## Как это работает

### 1. Создание таблицы (SheetsBlock)

- Пользователь нажимает **«Создать таблицу»**.
- Фронт отправляет `POST /sheets/create` с `userId`.
- Бэкенд вызывает **Google Apps Script** (URL и секрет в `.env`: `APPS_SCRIPT_URL`, `APPS_SCRIPT_SECRET`). Исходник скрипта: `scripts/google-apps-script/CreateTemplatesSheet.gs`. Скрипт создаёт таблицу с полной первой строкой заголовков (все колонки бэкапа), открывает доступ «Все, у кого есть ссылка» → **Редактор**, возвращает `editUrl`, `spreadsheetId`, `csvUrl`.
- Бэкенд сохраняет эту ссылку в `users.gsheet_url` и отдаёт её фронту.
- Пользователь видит ссылку «Ваша таблица: …» и может открыть таблицу в новой вкладке.
- Если у пользователя уже есть `gsheet_url`, кнопка «Создать таблицу» отключена (вторую таблицу не создаём).

**Важно:** без настроенных `APPS_SCRIPT_URL` и `APPS_SCRIPT_SECRET` создание таблицы вернёт ошибку. Развёртывание скрипта и настройка секрета описаны в `scripts/google-apps-script/README.md`.

### 2. Синхронизация шаблонов из таблицы (TemplatesSyncBlock)

- Пользователь нажимает **«Синхронизировать шаблоны»**.
- Фронт отправляет `POST /templates/sync` с `userId`.
- Бэкенд:
  1. Читает из БД `users.gsheet_url` для этого пользователя. Если пусто — возвращает ошибку «Вставь ссылку на Google Sheet».
  2. Преобразует ссылку на таблицу в URL экспорта в CSV:  
     `https://docs.google.com/spreadsheets/d/{id}/export?format=csv&gid={gid}`  
     (лист по умолчанию — `gid=0`, либо из ссылки).
  3. Скачивает CSV по этому URL (таблица должна быть доступна «по ссылке» хотя бы на просмотр).
  4. Парсит CSV (Papa.parse). Поддерживаются колонки полного снимка: **enabled**, **order**, **title**, **text**, **media_url**, **send_media_as_file**, **wa_speed_factor**, **tg_speed_factor**, **wa_default_send_time**, **tg_default_send_time**. Старые таблицы с только базовыми колонками тоже работают.
  5. Строки, где есть хотя бы `title` или `text`, попадают в список шаблонов. Для каждой строки задаётся `sheet_row` = номер строки в таблице (2, 3, 4…).
  6. Данные **upsert** в `message_templates` по ключу `(user_id, sheet_row)` со всеми прочитанными полями.

Шаблоны, созданные **вручную в кабинете** (в разделе «Шаблоны» дашборда), имеют `sheet_row >= 1_000_000`. Синхронизация из Google Sheet их **не трогает** — перезаписываются только строки с «маленьким» `sheet_row`, соответствующие строкам таблицы.

Итог: таблица — это «источник правды» для части шаблонов; синхронизация подтягивает из неё данные в БД перед рассылкой.

## Экспорт и импорт (бэкап)

- **GET /templates/export** — скачать все шаблоны пользователя в CSV (все колонки снимка). Ответ: `Content-Type: text/csv`, `Content-Disposition: attachment`.
- **POST /templates/import** — восстановить шаблоны из CSV. Тело: `{ "csv": "..." }` или multipart с полем `file` (файл .csv). Полная замена: все текущие шаблоны удаляются, из файла вставляются новые. В каждой строке обязательны **title** или **text**.
- Формат CSV и список колонок — в **docs/TEMPLATE-BACKUP-AND-SHEETS-DESIGN.md** (раздел «Формат CSV и шапка таблицы»).
