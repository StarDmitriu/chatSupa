# Срочно: настройка DNS для chattrassylka.ru

## Проблема

Certbot не может выдать SSL-сертификат, потому что домен **chatrassylka.ru** не резолвится (NXDOMAIN). Это значит, что DNS-записи для домена не настроены или указывают не на этот сервер.

## Что сделать СЕЙЧАС

### 1. Узнать IP этого сервера

```bash
curl ifconfig.me
# или
hostname -I
```

**Текущий IP сервера:** `45.12.75.57` (или проверьте выше)

### 2. Настроить DNS у регистратора домена

Зайдите в панель управления доменом (где куплен chattrassylka.ru) и добавьте:

**A-запись:**
- **Имя:** `@` (или пусто, для chattrassylka.ru)
- **Значение:** `45.12.75.57` (IP этого сервера)
- **TTL:** 3600 (или по умолчанию)

**A-запись для www:**
- **Имя:** `www`
- **Значение:** `45.12.75.57`
- **TTL:** 3600

### 3. Проверить DNS

После сохранения подождите 5–30 минут, затем проверьте:

```bash
nslookup chattrassylka.ru
# или
dig chattrassylka.ru A
```

Должен вернуться IP **45.12.75.57** (или IP вашего сервера).

### 4. Получить SSL-сертификат

Когда DNS настроен и резолвится:

```bash
sudo /var/www/scripts/get-ssl-cert.sh
```

Скрипт автоматически:
- Проверит DNS
- Получит сертификат Let's Encrypt
- Обновит конфиг nginx

После этого сайт будет доступен по **https://chatrassylka.ru/** с валидным SSL-сертификатом.

---

**Важно:** Пока DNS не настроен, сайт может быть недоступен или показывать страницу "Success! Your new web server is ready to use" (если запросы идут на другой сервер/прокси).
