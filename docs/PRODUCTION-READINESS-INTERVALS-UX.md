# Готовность к продакшену: интервалы и единый UX

## Что сделано

### 1. Единое понимание интервалов

- **Окно времени** (кнопка «Время» на странице «Рассылки») — в какие часы суток разрешена отправка. Подсказка в попапе уточнена.
- **Паузы рассылки** (кнопка «Паузы…»): в попапе переименовано в «Интервалы рассылки», добавлен текст, что здесь задаются только паузы для запуска; интервал по группе — в Группах WA/TG и в шаблоне.
- **Подписи в попапе «Паузы»**:
  - TG/WA: «пауза между отправками в разные группы (сек)»;
  - «Пауза между шаблонами» + пояснение «между волнами по шаблонам»;
  - «Повтор волны» + пояснение «через сколько минут снова отправить ту же волну».
- **Справка (?)**: в подсказке по разделу «Рассылки» добавлен блок «Интервалы и время — где что настраивать»: окно времени, пауза между группами, пауза между шаблонами, повтор волны, интервал по группе (Группы WA/TG, шаблон).

### 2. Интервал по группе в одном месте для WA и TG

- В **Группы WA** добавлена колонка «Интервал» (Select с `SEND_INTERVAL_OPTIONS`), по аналогии с Группами TG. API `POST /whatsapp/groups/time` уже был, использован тот же вызов при смене интервала.
- Итог: интервал (или фиксированное время) по группе задаётся в **Группы WA**, **Группы TG** или в настройках шаблона — одна логика, одно поле `send_time` на группу.

### 3. Логика без изменений

- Бэкенд: `time_from`/`time_to`, `between_groups_sec_*`, `between_templates_min_*`, `repeat_*`, `send_time` по группе — без изменений.
- Иерархия как в `INTERVALS-LOGIC-REPORT.md`: окно → пауза между группами → пауза между шаблонами → повтор волны; интервал по группе — отдельно в группах/шаблоне.

## Рекомендации на будущее

- **Один источник опций интервала**: ключи `GROUP_INTERVALS` (бэкенд) и `SEND_INTERVAL_OPTIONS` (фронт) держать в синхронизации; при добавлении варианта (например «раз в 8 ч») добавлять в оба места или вынести опции в API.
- **Фиксированное время (HH:mm)** в группах: в TG/WA в Select можно добавить опцию «Время (HH:mm)» с TimePicker, чтобы записывать в `send_time` значение вида `09:00` (бэкенд уже поддерживает).
