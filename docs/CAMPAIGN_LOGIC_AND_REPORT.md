# Логика рассылок: шаблоны, группы, запуск/остановка

## 1. Где можно остановить и запустить рассылку

**Остановка (Stop):**
- **Страница «Рассылки»** (`/dashboard/campaigns`) — кнопки «Остановить WA» и «Остановить TG».
- **Страница «Прогресс рассылки»** (`/dashboard/campaign?wa=...&tg=...`) — те же кнопки «Остановить WA» / «Остановить TG».

Оба вызывают `POST /campaigns/:campaignId/stop`. Pending/processing jobs помечаются как skipped.

**Запуск (Start):**
- **Страница «Рассылки»** — основной запуск: режим (TG/WA/both), окно времени, «Запустить рассылку».
- **Страница «Прогресс рассылки»** — добавлена кнопка **«Запустить рассылку»** (переход на `/dashboard/campaigns`), чтобы после остановки быстро перейти к запуску новой рассылки.

Requeue (повтор тех же jobs) есть в API (`POST /campaigns/:id/requeue`), в UI не вызывается.

## 2. Как устроены шаблоны и группы

**Два уровня выбора:**

1. **Группы «для рассылки вообще»** (разделы Группы WA / Группы TG):
   - Галочка «Выбрано» → `is_selected = true` (группа участвует в рассылках).
   - Поле «Интервал» → `send_time` (интервал в минутах, например `2-4h`, или фиксированное время HH:mm).

2. **Привязка шаблон → группы** (`template_group_targets`):
   - В **редактировании каждого шаблона** выбирают, в какие группы этот шаблон отправлять (отдельно WA и TG).
   - Если для шаблона заданы цели — в рассылку попадают только эти группы (из числа выбранных).
   - Если для шаблона целей нет, но хотя бы у одного шаблона они есть — этот шаблон не отправляется ни в одну группу.
   - Если ни у одного шаблона нет целей — все включённые шаблоны отправляются во все выбранные группы.

**Какие шаблоны попадают в рассылку:** только с `enabled = true`, порядок по полю `order`.

**Как создаются jobs при запуске:**
- Загружаются группы с `is_selected = true` и шаблоны с `enabled = true`.
- Для каждого шаблона берутся target-группы из `template_group_targets` (или все выбранные, если целей ни у кого нет).
- Для каждой пары (шаблон, группа) создаётся один job с `scheduled_at`:
  - без send_time: общий курсор в окне времени + betweenGroupsSec / betweenTemplatesMin;
  - send_time = интервал (2-4h, 6h, 24h и т.д.): следующая отправка в группу = последняя + случай из интервала;
  - send_time = HH:mm: фиксированное время в окне.

**Повтор волн (repeat):** если при создании кампании включён repeat, сервис по таймеру создаёт новую волну (те же шаблоны и группы) по `next_repeat_at`.

## 3. Сценарий: 5 сообщений в 25 групп, по 5 в каждую, интервал 4 ч, в течение суток

**Что нужно:**
- 5 шаблонов (enabled, порядок по order).
- 25 групп в одном канале (WA или TG), все с `is_selected = true`.
- У каждого из 5 шаблонов в настройках шаблона выбрать все 25 групп (цели для канала).

**Интервал 4 часа:** в системе есть `2-4h` (120–240 мин), нет ровно 4h. Ближайший вариант — «2–4 часа»; для ровно 4 ч можно добавить пресет `4h` (240 мин) в бэкенд `GROUP_INTERVALS` и во фронт `SEND_INTERVAL_OPTIONS`.

**Как будет работать:** окно времени 00:00–23:59; у всех 25 групп выставить интервал `2-4h` (или будущий `4h`). При запуске для каждого шаблона и каждой группы создаётся job; время в группе считается по правилу интервала (следующее сообщение в группу — через 2–4 ч после предыдущего). В каждую группу уйдёт 5 сообщений (по одному на шаблон), в течение суток.

## 4. Что не доделано для полноценной работы

**Уже есть:** start-multi, stop, progress, repeat, template_group_targets, send_time (интервал и фиксированное время на бэкенде), API requeue.

**Желательно:**
1. Добавить интервал ровно 4 ч: в бэкенде `GROUP_INTERVALS` и во фронте `SEND_INTERVAL_OPTIONS` опцию «4 часа» (`4h`).
2. В UI групп дать выбор фиксированного времени (HH:mm), не только интервалов — бэкенд уже поддерживает.
3. В UI дать кнопку «Запустить повторно» для остановленной/завершённой кампании (вызов requeue).
4. На фронте обрабатывать ошибку подписки (SubscriptionGuard) при start-multi/requeue и вести в кабинет/подписку.

---

## 5. Схема

```
Шаблоны (enabled, order, template_group_targets) + Группы (is_selected, send_time)
  → [Рассылки] timeFrom–timeTo, TG/WA/both, betweenGroups, betweenTemplates, repeat
  → POST start-multi → createWaveAndEnqueue → campaign_jobs → очередь → воркер → WA/TG
Остановка: POST stop → pending/processing → skipped
Повтор: по next_repeat_at создаётся новая волна
```

**Код:** campaigns — `backend/src/campaigns/campaigns.service.ts`, `campaigns.controller.ts`; UI — `frontend/.../campaigns/page.tsx`, `campaign/page.tsx`; цели шаблонов — `templates.service.ts`, `templates/[templateId]/page.tsx`; интервалы — `sendIntervals.ts`, `GROUP_INTERVALS` в campaigns.service.
