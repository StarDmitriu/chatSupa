# Почему «хостинг не принимает сайты» — из‑за DNS у регистратора

## Что видно сейчас

- **Сервер отдаёт сайт** — при запросе с `Host: chattrassylka.ru` ответ **200**, отдаётся приложение ЧатРассылка.
- **Домен не резолвится** — `chattrassylka.ru` даёт **NXDOMAIN** (как будто домена или его DNS-записей нет).

То есть проблема не в том, что «хостинг не принимает» сайт, а в том, что **с интернета до вашего сервера не доходят запросы**, потому что по имени chattrassylka.ru ни один DNS-сервер не знает IP вашего хостинга.

---

## Если вы удаляли что‑то у регистратора

### 1. Удалены только DNS-записи (домен ещё у вас)

Нужно заново настроить DNS у того, кто даёт зону (регистратор или хостинг):

- **A-запись** для **chattrassylka.ru** (или `@`) → **45.12.75.57**
- **A-запись** для **www.chattrassylka.ru** (или `www`) → **45.12.75.57**

Где править: панель управления доменом у регистратора (Reg.ru, Beget, Timeweb, nic.ru и т.п.) → раздел «DNS», «Управление зоной», «Делегирование» и т.п.

После сохранения подождать 5–30 минут (иногда до нескольких часов), затем проверить:

```bash
nslookup chattrassylka.ru
# или
dig chattrassylka.ru A
```

Должен вернуться IP **45.12.75.57**.

---

### 2. Домен удалён/отвязан у регистратора

Если домен удалили, передали другому регистратору или сменили владельца:

- Нужно снова **привязать домен к хостингу**: в панели хостинга (Beget/Hestia и т.п.) указать, что chattrassylka.ru обслуживается этим сервером (добавить сайт/домен, если просят).
- И снова настроить DNS у того, кто сейчас управляет зоной (регистратор или DNS-хостинг): A-записи **@** и **www** → **45.12.75.57**.

Без корректных DNS-записей у регистратора хостинг «не принимает» сайты в том смысле, что к нему просто не приходят запросы по имени chattrassylka.ru.

---

### 3. DNS ведётся на Beget/хостинге

Если NS-серверы домена указывают на Beget (или ваш хостинг), то A-записи нужно править **в панели хостинга** (раздел «Домены» / «DNS»), а не у регистратора. Регистратор в этом случае только хранит NS, а сами A-записи — у хостинга.

---

## Итог

- **Хостинг готов**: nginx и приложение отдают сайт по chattrassylka.ru.
- **Чтобы сайт открывался из интернета**, у домена chattrassylka.ru (у того, кто управляет зоной) должны быть A-записи на **45.12.75.57**.
- Если вы что‑то удаляли у регистратора — восстановите A-записи (и при необходимости привязку домена к хостингу), после обновления DNS сайт начнёт открываться.

Проверка с вашего ПК после настройки DNS:

```bash
nslookup chattrassylka.ru
ping chattrassylka.ru
```

Должны быть ответы с IP **45.12.75.57**.
