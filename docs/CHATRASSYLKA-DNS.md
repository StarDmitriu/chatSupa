# Почему chattrassylka.ru показывает «Success! Your new web server is ready to use»

Эта страница появляется, когда запросы к домену **не попадают на этот сервер** (45.12.75.57), а идут на другой хостинг/прокси (часто — дефолтная страница панели при добавлении домена).

## Что сделать

1. **Проверить DNS**  
   У домена **chatrassylka.ru** (и при необходимости **www.chattrassylka.ru**) в зоне должна быть **A-запись** на IP этого сервера:
   - **45.12.75.57**

   Проверка с вашего ПК:
   ```bash
   nslookup chattrassylka.ru
   ```
   В ответе должен быть адрес **45.12.75.57**.

2. **Где править DNS**  
   В панели регистратора домена (где куплен chattrassylka.ru) или в панели Beget (если DNS ведётся там):  
   - Тип: **A**  
   - Имя: **@** (для chattrassylka.ru) и при необходимости **www** (для www.chattrassylka.ru)  
   - Значение: **45.12.75.57**

3. **Подождать обновления DNS**  
   После сохранения изменений подождать 5–30 минут (иногда до нескольких часов), затем обновить страницу (лучше в режиме инкогнито или с Ctrl+F5).

4. **Проверка с сервера**  
   На самом сервере приложение уже отдаётся: запрос с заголовком `Host: chattrassylka.ru` к этому серверу возвращает лендинг «ЧатРассылка». Как только DNS начнёт указывать на 45.12.75.57, то же самое увидите и в браузере по адресу http://chatrassylka.ru/

---

**localhost:58213** — это порт превью Cursor/IDE. Стили там могут не грузиться, потому что статика (`/_next/static/...`) рассчитана на тот же хост. Открывайте сайт по реальному адресу: **http://chatrassylka.ru/** (после настройки DNS) или **http://localhost:3001** при локальной разработке (`npm run dev` в frontend).
