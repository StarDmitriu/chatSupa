# Бэкапы шаблонов и снимки в Google Таблице

## Идея

Сейчас: создаётся пустая таблица, пользователь вручную заполняет колонки (enabled, order, title, text, media_url), потом «Синхронизировать» подтягивает данные в приложение. Это скорее «источник шаблонов из таблицы».

Предложение: переориентировать на **бэкап/снимок**:

- Клиент **сохраняет** свои шаблоны в один снимок (файл или таблица).
- Все поля шаблона участвуют в формате — **полный снимок**, все столбцы заданы.
- Снимок можно **открыть/восстановить** — загрузить из файла или таблицы в приложение.
- В перспективе: шаблонные рассылки для других пользователей, автоматический запуск кампаний, монетизация (продажа «готовых наборов»).

---

## Полный набор полей шаблона (message_templates)

Для бэкапа и формата таблицы нужны все данные, которые хранятся по шаблону (кроме `id`, `user_id`, `updated_at` при экспорте можно включить для информации).

| Колонка в таблице/CSV | Тип | Обязательность при импорте | Описание |
|----------------------|-----|----------------------------|----------|
| **enabled** | bool | нет (по умолчанию true) | Включён ли шаблон |
| **order** | int | нет (по умолчанию 1) | Порядок отправки |
| **title** | string | да | Название шаблона |
| **text** | string | да* | Текст сообщения (*хотя бы одно из title/text) |
| **media_url** | string | нет | Ссылка на медиафайл |
| **send_media_as_file** | bool | нет (false) | Отправлять медиа как файл |
| **wa_speed_factor** | int 10–400 | нет (100) | Множитель скорости для WA, % |
| **tg_speed_factor** | int 10–400 | нет (100) | Множитель скорости для TG, % |
| **wa_default_send_time** | string | нет | Интервал/время для WA (напр. 2-5m, 4h, 09:00) |
| **tg_default_send_time** | string | нет | Интервал/время для TG |

Для бэкапа «все поля обязательны к заполнению» = в **формате снимка** все эти колонки присутствуют; при импорте пустые трактуем как дефолты (как в таблице выше). Жёстко обязательны к непустому значению только **title** и **text** (хотя бы один).

## Сценарии

### 1. Сохранить бэкап (экспорт)

- **Вариант A — скачать CSV:** кнопка «Скачать бэкап шаблонов». Бэкенд отдаёт CSV со всеми колонками из списка выше (заголовок + строки по текущим шаблонам пользователя). Пользователь сохраняет файл — это и есть снимок.
- **Вариант B — сохранить в Google Таблицу:** по нажатию «Сохранить бэкап в таблицу» бэкенд отдаёт данные, фронт (или бэкенд через Apps Script) создаёт новую таблицу с заголовками и заполненными строками. Таблица = готовый снимок, который можно потом «открыть» (импорт по ссылке).

Реализация: сначала достаточно **Варианта A** (эндпоинт экспорта в CSV), без изменений Apps Script. Вариант B — отдельный шаг (расширение Apps Script или вызов API Google Sheets с сервера).

### 2. Создать таблицу под бэкап (шаблон таблицы)

- Вместо «просто пустая таблица» — создаётся таблица с **готовой шапкой**: все колонки из списка выше. Пользователь видит, что именно нужно заполнять, и может заполнить строки, затем нажать «Восстановить из таблицы» (импорт).
- Логика: тот же Apps Script при создании таблицы создаёт первый лист с первой строкой = заголовки (enabled, order, title, text, media_url, send_media_as_file, wa_speed_factor, tg_speed_factor, wa_default_send_time, tg_default_send_time). Опционально вторая строка — пример с подсказками (например, true, 1, Пример, Текст сообщения, , false, 100, 100, , ).

Это даёт «нужное количество столбцов в соответствии со всеми параметрами шаблонов» и ощущение «снимок, который можно сохранить и потом открыть».

### 3. Восстановить из бэкапа (импорт)

- Из файла: пользователь загружает CSV (или вставляет ссылку на опубликованный в просмотр лист) — бэкенд парсит CSV и применяет те же правила, что и при «Синхронизации», но с полным набором колонок.
- Из таблицы: как сейчас «Синхронизировать шаблоны», но:
  - читаем все колонки из списка выше;
  - валидация: в каждой строке обязательно заполнены **title** или **text** (или оба); остальное — по умолчанию при пустоте;
  - запись в БД с полными полями (send_media_as_file, wa_speed_factor, tg_speed_factor, wa_default_send_time, tg_default_send_time).

Текущий `syncFromSheet` сейчас заполняет только enabled, order, title, text, media_url; его нужно расширить до полного набора и при импорте считать «снимком» (полный набор колонок).

## Формат CSV и шапка таблицы (реализовано)

**Порядок колонок** (первая строка CSV / первая строка листа в Google Таблице):

```
enabled,order,title,text,media_url,send_media_as_file,wa_speed_factor,tg_speed_factor,wa_default_send_time,tg_default_send_time
```

| Колонка | Пример | Описание |
|---------|--------|----------|
| enabled | true / false | Включён ли шаблон |
| order | 1 | Порядок отправки |
| title | Название | Название шаблона |
| text | Текст сообщения | Тело сообщения (можно с переносами) |
| media_url | https://... | Ссылка на медиа (пусто — без медиа) |
| send_media_as_file | false | Отправлять медиа как файл |
| wa_speed_factor | 100 | Множитель скорости WA (10–400) |
| tg_speed_factor | 100 | Множитель скорости TG (10–400) |
| wa_default_send_time | 2-5m или 09:00 | Интервал/время для WA |
| tg_default_send_time | 4h | Интервал/время для TG |

При **импорте** (файл или таблица) обязательны к заполнению в каждой строке только **title** или **text** (или оба). Остальные поля могут быть пустыми — подставятся значения по умолчанию (enabled=true, order=1, send_media_as_file=false, wa_speed_factor=100, tg_speed_factor=100, время — пусто).

**Где доступно:**

- **Дашборд → Шаблоны:** блок «Снимок шаблонов (бэкап)»: Скачать бэкап, Восстановить из файла, Восстановить из таблицы.
- **Кабинет → Интеграции:** Таблица шаблонов (создать таблицу), Скачать бэкап, Восстановить из таблицы.

## Будущее: монетизация и авто-кампании

- **Шаблонные наборы (template packs):** вы или партнёры готовите снимок (CSV или публичная таблица) с готовыми шаблонами. Пользователь по ссылке «открывает» набор — импортирует шаблоны к себе. Можно продавать доступ к таким наборам или давать бесплатно как примеры.
- **Один клик — кампания:** снимок может быть расширен до «рассылка под ключ»: не только шаблоны, но и (в будущем) настройки времени, каналов и т.д. Пользователь нажимает «Запустить по этому снимку» — создаётся кампания с заданными шаблонами. Монетизация: платные «готовые рассылки» или подписка на каталог снимков.
- **Ограничения:** для платных снимков понадобится проверка подписки, лимиты на импорт, возможно отдельная сущность «template_pack» с метаданными и ценой.
