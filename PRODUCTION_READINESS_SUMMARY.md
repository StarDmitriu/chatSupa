# Готовность к продакшену — сводка по функционалу

**Дата:** 19.02.2026 (актуализация после PRODUCTION_READINESS_REPORT.md и правок)

---

## 1. Функционал — что готово

| Область | Статус | Комментарий |
|--------|--------|-------------|
| **Auth** | ✅ | Вход по SMS (при заданном SMSRU_API_ID), JWT, регистрация, профиль, рефералы |
| **Подписки** | ✅ | Trial, платный период, SubscriptionGuard на запуск рассылки и requeue |
| **Кабинет** | ✅ | Подписка, поддержка, шаблоны, группы, рассылки |
| **Шаблоны** | ✅ | CRUD, медиа, привязка к группам (TG/WA), порядок; колонка send_media_as_file с fallback при её отсутствии в БД |
| **Группы WA/TG** | ✅ | Синк, выбор, пагинация, лимит 50k при создании волны рассылки |
| **Рассылки** | ✅ | Старт/стоп, прогресс, повтор волн (CAMPAIGN_REPEAT_*), лимит 1000 снят (до 50k заданий) |
| **Очередь** | ✅ | BullMQ + Redis, воркер campaign-send, TG/WA роутинг |
| **Оплата (Prodamus)** | ✅ | Создание платежа, webhook, продление подписки; при отсутствии PRODAMUS_* — ошибка при обращении |
| **Лиды** | ✅ | Форма на лендинге → Supabase lead_requests |
| **Админка** | ✅ | Список пользователей, блок, trial/paid ± дни; доступ по JWT + is_admin |
| **Telegram** | ✅ | QR-логин, 2FA, группы, отправка в кампаниях; при отсутствии TG_* — не работает |
| **WhatsApp** | ✅ | QR, группы, отправка; сессии на диске (wa_auth) |
| **Sheets** | ⚠️ Опционально | При APPS_SCRIPT_* — создание таблицы для пользователя; без них — ошибка при вызове |

Итог: **ядро для прода полностью реализовано** — регистрация, подписка (trial/оплата), шаблоны, группы, рассылки (TG/WA), лиды, админка.

## 2. Зависимости от окружения

| Переменная / сервис | Обязательно для прода? | Если нет |
|---------------------|------------------------|----------|
| SUPABASE_*, JWT_SECRET | ✅ Да | Backend не стартует / нет авторизации |
| REDIS (URL/host/port) | ✅ Да | Очередь рассылок не работает |
| SMSRU_API_ID | ✅ Да (для входа по SMS) | Коды только в лог, пользователи не войдут |
| TG_API_ID, TG_API_HASH | ✅ Да (если нужен Telegram) | Telegram (QR, группы, рассылки) не работает |
| PRODAMUS_* | ✅ Да (если нужна оплата) | Ошибка при создании платежа |
| APPS_SCRIPT_* | Нет | Только интеграция с Google Sheets отключена |

Проверка: в `backend/.env` должны быть заданы все нужные для вашего сценария переменные (см. ENV_CHECK_REPORT.md).

## 3. Что доработать / риски (не блокеры)

| Пункт | Важность | Действие |
|-------|----------|----------|
| **Логи Prodamus** | Средняя | В webhook есть `console.log('sign:', signature)` и др. — лучше перевести на логгер и не логировать подпись/чувствительные поля |
| **Логи auth/queue** | Низкая | Часть `console.error`/`console.warn` — по возможности заменить на Nest Logger |
| **SEO** | Низкая | В layout `lang='en'` при русском контенте; нет title/description по страницам, robots.txt, sitemap (как в отчёте 07.02) |
| **Шрифт** | Низкая | Manrope только `latin` — при желании добавить кириллицу |
| **Колонка send_media_as_file** | Опционально | В логах может быть «column send_media_as_file does not exist» — в коде есть fallback; для полной функции — миграция в Supabase |
| **Миграции БД** | По чеклисту | DEPLOYMENT_CHECKLIST.md: индексы и скрипт по дубликатам групп — выполнить в Supabase при необходимости |

## 4. Безопасность (кратко)

- Секреты только в env, не в коде; .env в .gitignore.
- Админка защищена AdminGuard (is_admin в БД).
- Запуск рассылки и requeue защищены SubscriptionGuard.
- Рекомендации из ROOT-CAUSE-AND-FIX.md и security-audit.sh: обновление Next.js (fix-react2shell-next), ufw, Fail2Ban, смена секретов после инцидента.

## 5. Вывод

**По функционалу для прода всё готово:** авторизация, подписки, шаблоны, группы, рассылки (TG/WA), оплата, лиды, админка работают при корректном backend/.env и Redis.  

Оставшиеся пункты — логи (Prodamus и др.), SEO, шрифт, опциональные миграции и колонка send_media_as_file — не блокируют запуск, их можно закрывать по приоритету после выката.
