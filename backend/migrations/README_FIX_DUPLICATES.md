# Исправление дубликатов групп

## Проблема
Группы могут появляться дважды в списке из-за дубликатов в базе данных. Это может происходить если:
- Отсутствует уникальный индекс на `(user_id, wa_group_id)` или `(user_id, tg_chat_id)`
- Произошла ошибка при синхронизации групп
- Данные были вставлены вручную или через другой процесс

## Решение

### Шаг 1: Проверка дубликатов
Выполните первые два запроса из `fix_duplicate_groups.sql` чтобы увидеть, какие группы дублируются:

```sql
-- Для WhatsApp
SELECT user_id, wa_group_id, COUNT(*) as count
FROM whatsapp_groups
GROUP BY user_id, wa_group_id
HAVING COUNT(*) > 1;

-- Для Telegram
SELECT user_id, tg_chat_id, COUNT(*) as count
FROM telegram_groups
GROUP BY user_id, tg_chat_id
HAVING COUNT(*) > 1;
```

### Шаг 2: Удаление дубликатов
Выполните весь скрипт `fix_duplicate_groups.sql` в Supabase SQL Editor. Скрипт:
1. Найдет все дубликаты
2. Удалит старые записи, оставив только самую свежую (по `updated_at DESC`)
3. Создаст уникальные индексы для предотвращения будущих дубликатов

### Шаг 3: Проверка
После выполнения скрипта проверьте, что дубликатов больше нет:

```sql
SELECT 'WhatsApp duplicates:', COUNT(*) 
FROM (
    SELECT user_id, wa_group_id, COUNT(*) as cnt
    FROM whatsapp_groups
    GROUP BY user_id, wa_group_id
    HAVING COUNT(*) > 1
) t;
```

## Важно
- **Сделайте бэкап базы данных** перед выполнением скрипта удаления
- Скрипт оставляет самую свежую запись (по `updated_at DESC`)
- После создания уникальных индексов новые дубликаты будут предотвращены на уровне БД

## Продакшен
1. В Supabase SQL Editor выполните **весь** файл `fix_duplicate_groups.sql` (удаление дубликатов + создание уникальных индексов).
2. Проверка после применения: в конце скрипта есть запросы — оба счётчика дубликатов должны быть `0`.
3. Опционально: `node migrations/test_duplicates.js` (нужны `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY` в `.env`) — выведет, есть ли ещё дубликаты.

При отдаче списка групп API уже дедуплицирует ответ (для Telegram при дублях выбирается запись с аватаром).

## Логирование
После применения исправлений, бэкенд будет логировать более детальную информацию о найденных дубликатах, включая ID записей и названия групп для упрощения отладки.
