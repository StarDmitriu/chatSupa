# Деплой и кэш: 404 на чанки, 502 на API

## 404 на `/_next/static/chunks/...` (css/js)

**Причина:** Браузер или CDN отдаёт **старую HTML/RSC**, в которой прописаны старые имена чанков (например `e9fa27303fa2bbca.css`). После новой сборки Next.js генерирует **новые хеши** в именах файлов, старые файлы в билде уже не лежат → 404.

**Что делать:**

1. **После каждого деплоя фронта** — полная пересборка и выкладка всего `frontend/.next/standalone` (включая `.next/static`).
2. **Кэш HTML:** не кэшировать ответы страниц (HTML/RSC) долго. На nginx для маршрутов вида `/`, `/cabinet`, `/dashboard/:path*` можно выставить `Cache-Control: no-cache` или `max-age=0`, чтобы пользователи получали свежую разметку со следующими запросами.
3. **Пользователь:** при «битой» странице после обновления сайта — жёсткое обновление (Ctrl+Shift+R / Cmd+Shift+R) или очистка кэша для сайта.

Статика `/_next/static/*` в `next.config.ts` отдаётся с `Cache-Control: immutable` — после того как пользователь получил актуальную HTML, чанки кэшируются надолго.

## 502 Bad Gateway на `/api/...`

**Причина:** Запрос идёт в Next.js → rewrite на бэкенд. 502 значит, что бэкенд не ответил (не запущен, упал, таймаут, неверный `BACKEND_INTERNAL_URL`).

**Что проверить:**

- Запущен ли backend (pm2/docker), слушает ли порт.
- В окружении фронта (или в next) задан ли `BACKEND_INTERNAL_URL` на доступный из процесса Next адрес бэкенда (например `http://localhost:3000` или внутренний хост в Docker).
- Логи бэкенда и nginx (если есть) на момент 502.

## 404 на `/api/telegram/groups/...`, `/api/telegram/premium-status/...`

Роуты в бэкенде есть (`GET telegram/groups/:userId`, `GET telegram/premium-status/:userId`). Если в логах приходит 404:

- Убедиться, что запрос реально уходит на бэкенд (rewrite не режет путь).
- Если одновременно видите 502 на других запросах — сначала устранить недоступность бэкенда; 404 могут быть следствием нестабильной работы или кэша ошибок.

Фронт при ошибке/404 на `premium-status` и части запросов к группам использует fallback (например лимит 2GB для файлов), страница не должна «падать».
