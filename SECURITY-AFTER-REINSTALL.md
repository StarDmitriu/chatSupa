# Чеклист после переустановки VPS (защита от повторного заражения)

**Контекст:** Сервер многократно заражался криптомайнером (процессы вида `fbhichqybyld`, `cjdiioqfnyja`, запуск через `nohup /etc/<случайное_имя>`). Возможный вектор — уязвимость React2Shell в Next.js (CVE-2025-66478) или другое веб-приложение.

---

## 1. Сразу после переустановки (до деплоя приложений)

- [ ] **Обновить ОС:** `apt update && apt upgrade -y`
- [ ] **Файрвол:** включить и разрешить только нужные порты:
  ```bash
  ufw allow 22/tcp   # SSH
  ufw allow 80/tcp   # HTTP (если нужен)
  ufw allow 443/tcp # HTTPS
  ufw enable
  ```
- [ ] **Fail2Ban** (защита SSH от брутфорса):
  ```bash
  apt install fail2ban -y
  systemctl enable --now fail2ban
  ```
- [ ] **Проверка целостности бинарников** (после любых подозрений):
  ```bash
  debsums -s 2>/dev/null || true   # если установлен debsums
  ```

---

## 2. Перед деплоем веб-приложения (Next.js / Node)

- [ ] **Обновить Next.js до безопасной версии** (если используется App Router):
  ```bash
  cd /path/to/your/app
  npx fix-react2shell-next
  # или вручную: npm install next@15.0.5 (или актуальная патч-версия)
  ```
- [ ] **Сменить все секреты** (пароли БД, API-ключи, env), если сервер ранее был заражён.
- [ ] Не деплоить приложение с уязвимыми зависимостями — сервер будет заражаться снова.

---

## 3. Регулярная проверка (аудит)

Запускать периодически или после деплоя:

```bash
sudo /var/www/security-audit.sh
```

Или вручную:

- [ ] **Подозрительные процессы:** `ps aux | grep -E '%CPU|nohup|/etc/[a-z]\{10,\}'`
- [ ] **Файлы в /etc с случайными именами:** `ls -la /etc/` — ничего вида `cjdiioqfnyja`, `fbhichqybyld`.
- [ ] **Крон:** `grep -r "nohup\|/etc/[a-z]\{8,\}" /etc/cron* /var/spool/cron 2>/dev/null` — должно быть пусто.
- [ ] **Системные утилиты на месте:** `which ls python3` — должны быть `/usr/bin/ls`, `/usr/bin/python3`.

---

## 4. Если снова видите высокую нагрузку или странный процесс

1. **Не перезагружать сразу** — сохранить вывод для анализа:
   ```bash
   ps aux > /tmp/ps_snapshot.txt
   ls -la /etc/ > /tmp/etc_list.txt
   ```
2. **Завершить подозрительный процесс:** `kill -9 <PID>`
3. **Удалить бинарник**, если он в `/etc/` или в другом нестандартном месте.
4. **Проверить крон и systemd** на задачи, перезапускающие этот процесс.
5. **Устранить точку входа** (обновить уязвимое ПО, закрыть порты, сменить пароли).
6. При необходимости — снова переустановить ОС и пройти этот чеклист с пункта 1.

---

## 5. Полезные инструменты (по желанию)

- **ClamAV:** `apt install clamav && freshclam && clamscan -r /home /var/www`
- **rkhunter / chkrootkit:** поиск руткитов после инцидента.
